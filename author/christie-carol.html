git merge

This is a very simple list of instructions on how to do a basic git merge. When you're doing a git merge, you're working with two branches.

Current Branch
The branch where HEAD is pointing, which will get changed by the merge.
Target Branch
Remains unchanged after the merge, it is the one you want to keep the same.

You can check to see where the HEAD is pointing using: 

    git rev-parse --short HEAD

You should switch to the branch you want to change using:

    git checkout <branch-name>

To make sure you have the most recent changes to the remote, do:

    git fetch
    git pull

on the master branch.

Finally, do 

    git merge <target-branch>

to get the changes from the target branch onto the current branch.
